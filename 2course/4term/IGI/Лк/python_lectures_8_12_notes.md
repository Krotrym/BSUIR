## Лекция 8. ООП и классы в Python

### 1. Класс и объект

Класс - это шаблон, описание некоторого типа объектов. Он задает, какие данные объект может хранить и какие действия он умеет выполнять.

Объект, или экземпляр класса, - конкретная сущность, созданная по этому шаблону.

Пример:

```python
class Student:
    pass

a = Student()
```

Здесь `Student` - класс, а `a` - объект класса `Student`.

В Python почти все является объектом: числа, строки, списки, функции, классы. Например:

```python
"hello world".count("l")
```

Строка `"hello world"` - объект класса `str`, а `count` - его метод.

### 2. Поля, методы и `self`

Атрибуты объекта - данные, связанные с объектом.

Методы - функции, определенные внутри класса. Первый параметр обычного метода почти всегда называют `self`. Он указывает на конкретный объект, с которым сейчас работает метод.

```python
class Student:
    def say_hello(self):
        print("Hello")

s = Student()
s.say_hello()
```

Когда вызывается `s.say_hello()`, Python сам передает объект `s` внутрь метода как `self`.

### 3. Статические и динамические атрибуты

В Python атрибуты можно создавать динамически, то есть уже после создания объекта:

```python
class A:
    pass

a = A()
a.x = 10
```

Атрибут `x` появился только у объекта `a`. Это удобно, но может приводить к хаосу, если поля создаются где попало.

Атрибуты класса задаются внутри класса и общие для класса:

```python
class User:
    role = "guest"
```

Если атрибут читается через объект, Python сначала ищет его у объекта, потом у класса.

Внутренний словарь объекта обычно доступен через `__dict__`:

```python
print(a.__dict__)
```

Он показывает динамические атрибуты объекта.

### 4. Конструктор `__init__`

`__init__` вызывается сразу после создания объекта и обычно используется для начальной настройки объекта:

```python
class Student:
    def __init__(self, name, age):
        self.name = name
        self.age = age

s = Student("Ann", 19)
```

В Python нет классической перегрузки методов по сигнатурам, как в C++/Java. Нельзя объявить несколько `__init__` с разными наборами параметров. Используют значения по умолчанию, `*args`, `**kwargs` или альтернативные конструкторы через `@classmethod`.

```python
class Point:
    def __init__(self, x=0, y=0):
        self.x = x
        self.y = y
```

### 5. `__new__`

`__new__` отвечает за само создание объекта, а `__init__` - за его инициализацию.

Обычно `__new__` не трогают. Он нужен при создании неизменяемых типов, singleton-подобных решений или сложного метапрограммирования.

```python
class A:
    def __new__(cls, *args, **kwargs):
        obj = super().__new__(cls)
        return obj

    def __init__(self):
        print("init")
```

### 6. Деструктор `__del__`

`__del__` вызывается перед уничтожением объекта сборщиком мусора:

```python
class A:
    def __del__(self):
        print("object deleted")
```

Важно: в Python нельзя точно полагаться на момент вызова `__del__`. Для файлов, соединений и других ресурсов лучше использовать `with` и менеджеры контекста.

### 7. Объект как функция: `__call__`

Если определить `__call__`, объект можно вызывать как функцию:

```python
class Counter:
    def __init__(self):
        self.value = 0

    def __call__(self):
        self.value += 1
        return self.value

c = Counter()
print(c())
print(c())
```

Так часто делают объекты-обработчики, валидаторы, декораторы, фабрики.

### 8. Область видимости в классе

В теле класса создается пространство имен класса. В методах локальные переменные живут отдельно. Чтобы обратиться к полю объекта, нужен `self`.

```python
class A:
    x = 1

    def method(self):
        y = 2
        print(self.x)
```

`x` - атрибут класса, `y` - локальная переменная метода.

### 9. Инкапсуляция

Инкапсуляция - идея скрывать внутреннее устройство объекта и давать управляемый интерфейс для работы с ним.

В Python нет жестких `private`, как в Java/C++. Есть соглашения:

`name` - обычный публичный атрибут.

`_name` - внутренний атрибут: "лучше не трогать снаружи".

`__name` - name mangling: Python переименует атрибут в `_ClassName__name`, чтобы уменьшить риск случайного конфликта в наследовании.

Инкапсуляция нужна не для "запрета ради запрета", а чтобы объект сохранял корректное состояние.

### 10. Наследование

Наследование позволяет создать класс на основе другого класса.

```python
class Animal:
    def speak(self):
        print("...")

class Dog(Animal):
    def speak(self):
        print("woof")
```

`Dog` наследует поведение `Animal`, но может его переопределить.

Синтаксис множественного наследования:

```python
class Child(Parent1, Parent2):
    pass
```

При поиске методов Python использует MRO - Method Resolution Order, порядок разрешения методов. Посмотреть его можно так:

```python
print(Child.mro())
```

### 11. `super()`

`super()` позволяет вызвать метод родительского класса согласно MRO.

```python
class Person:
    def __init__(self, name):
        self.name = name

class Student(Person):
    def __init__(self, name, group):
        super().__init__(name)
        self.group = group
```

Это важно при наследовании, чтобы не копировать код родителя и корректно работать с множественным наследованием.

### 12. Полиморфизм

Полиморфизм - возможность работать с разными объектами через одинаковый интерфейс.

```python
class Cat:
    def speak(self):
        return "meow"

class Dog:
    def speak(self):
        return "woof"

for animal in [Cat(), Dog()]:
    print(animal.speak())
```

Код вызывает `speak`, не проверяя конкретный класс объекта.

### 13. Утиная типизация

Python часто использует "утиную типизацию": если объект ведет себя как нужный тип, его можно использовать.

Главная мысль: важен не класс объекта, а наличие нужных методов и атрибутов.

```python
def print_length(obj):
    print(len(obj))

print_length("abc")
print_length([1, 2, 3])
```

Функции все равно, строка это или список. Главное, что объект поддерживает `len`.

### 14. Специальные, или магические, методы

Методы вида `__имя__` Python использует для встроенных операций.

Частые методы:

`__init__` - инициализация объекта.

`__str__` - строка для пользователя, вызывается `str(obj)` и `print(obj)`.

`__repr__` - техническое представление объекта, полезно для отладки.

`__eq__` - сравнение через `==`.

`__lt__`, `__le__`, `__gt__`, `__ge__` - сравнения `<`, `<=`, `>`, `>=`.

`__add__`, `__sub__`, `__mul__`, `__truediv__` - арифметические операции.

`__len__` - `len(obj)`.

`__getitem__`, `__setitem__` - индексация и срезы.

`__iter__`, `__next__` - итерация.

Пример перегрузки оператора:

```python
class Vector:
    def __init__(self, x, y):
        self.x = x
        self.y = y

    def __add__(self, other):
        return Vector(self.x + other.x, self.y + other.y)

    def __repr__(self):
        return f"Vector({self.x}, {self.y})"

print(Vector(1, 2) + Vector(3, 4))
```

### 15. Композиция и агрегирование

Композиция - объект содержит внутри себя другие объекты.

```python
class Engine:
    def start(self):
        print("engine started")

class Car:
    def __init__(self):
        self.engine = Engine()
```

Это отношение "имеет": машина имеет двигатель.

Наследование - отношение "является": студент является человеком. Композиция часто безопаснее наследования, потому что меньше связывает классы.

### 16. Геттеры, сеттеры и `property`

Прямой доступ к атрибутам иногда опасен:

```python
user.age = -100
```

Свойства позволяют выглядеть как обычный атрибут, но внутри выполнять код: проверку, вычисление, логирование.

```python
class User:
    def __init__(self, age):
        self.age = age

    @property
    def age(self):
        return self._age

    @age.setter
    def age(self, value):
        if value < 0:
            raise ValueError("age must be non-negative")
        self._age = value
```

`property(fget, fset, fdel, doc)` - встроенный механизм создания управляемого атрибута. Декоратор `@property` - удобная форма записи.

Свойство может возвращать вычисляемое значение:

```python
class Rectangle:
    def __init__(self, w, h):
        self.w = w
        self.h = h

    @property
    def area(self):
        return self.w * self.h
```

### 17. Типы методов

Обычный метод экземпляра принимает `self` и работает с конкретным объектом.

```python
def method(self):
    ...
```

Метод класса принимает `cls` и работает с классом. Часто используется как альтернативный конструктор.

```python
class User:
    def __init__(self, name):
        self.name = name

    @classmethod
    def anonymous(cls):
        return cls("anonymous")
```

Статический метод не принимает ни `self`, ни `cls`. Это функция, логически лежащая внутри класса.

```python
class Math:
    @staticmethod
    def add(a, b):
        return a + b
```

### 18. Метаклассы и `type`

Метакласс - класс, экземплярами которого являются классы.

Обычная цепочка:

```python
obj -> class -> metaclass
```

В Python классы тоже объекты. Обычно класс является объектом метакласса `type`.

`type` можно использовать как функцию:

```python
type(123)  # int
```

И как конструктор классов:

```python
A = type("A", (), {"x": 10})
```

Это примерно равно:

```python
class A:
    x = 10
```

Формат:

```python
type(name, bases, dct)
```

`name` - имя класса.

`bases` - кортеж родительских классов.

`dct` - словарь атрибутов и методов.

Метаклассы используют, когда нужно контролировать создание классов: автоматически регистрировать подклассы, проверять структуру классов, добавлять атрибуты, декорировать методы, строить декларативные API.

Пример:

```python
class Meta(type):
    def __new__(mcls, name, bases, namespace):
        namespace["created_by_meta"] = True
        return super().__new__(mcls, name, bases, namespace)

class A(metaclass=Meta):
    pass

print(A.created_by_meta)
```

Метаклассы - мощный, но редкий инструмент. В обычном коде чаще хватает декораторов, наследования и композиции.

### 19. Примеси, или mixin

Mixin - класс-примесь, который добавляет небольшое поведение через множественное наследование, но не является главным родителем.

```python
class JsonMixin:
    def to_json(self):
        import json
        return json.dumps(self.__dict__)

class User(JsonMixin):
    def __init__(self, name):
        self.name = name
```

Примеси обычно:

- маленькие;
- отвечают за одну возможность;
- не должны сильно зависеть от внутренностей основного класса;
- не должны создавать сложную иерархию.

## Лекция 9. Файлы и сериализация данных

### 1. Что такое файл

Файл - именованная область данных на носителе. Он хранится в файловой системе.

У файла есть свойства:

- имя и расширение;
- путь;
- даты создания, изменения, доступа;
- владелец;
- атрибуты и права доступа.

Пути бывают абсолютные и относительные.

Абсолютный путь указывает место от корня файловой системы:

```text
C:\user\python\example.py
/home/user/python/example.py
```

Относительный путь считается от текущего рабочего каталога:

```text
example.py
folder/example.py
```

### 2. Операции с файлами

Операции с открытым файлом: открыть, прочитать, записать, переместиться по позиции, закрыть.

Операции без открытия файла: переименовать, удалить, скопировать, проверить существование, получить размер и даты.

### 3. `open`

Файл открывается функцией:

```python
open(file, mode="r", encoding=None, newline=None)
```

Главные параметры:

`file` - путь к файлу.

`mode` - режим открытия.

`encoding` - кодировка для текстового режима, например `utf-8`.

`newline` - правила обработки переводов строк.

Если файл не удалось открыть, возникает `OSError` или его наследники, например `FileNotFoundError`.

### 4. Режимы открытия

`r` - чтение, файл должен существовать.

`w` - запись, файл будет создан или очищен.

`x` - создать новый файл, ошибка если он уже существует.

`a` - дозапись в конец файла.

`+` - чтение и запись.

`t` - текстовый режим.

`b` - бинарный режим.

Примеры:

```python
open("data.txt", "r", encoding="utf-8")
open("data.txt", "w", encoding="utf-8")
open("image.png", "rb")
open("out.bin", "wb")
open("data.txt", "a", encoding="utf-8")
```

### 5. Методы файлового объекта

Свойства:

`name` - имя файла.

`mode` - режим открытия.

`encoding` - кодировка.

`closed` - закрыт ли файл.

Методы:

`close()` - закрыть файл.

`read(count)` - прочитать `count` символов/байт или все до конца.

`readline(count)` - прочитать одну строку.

`readlines()` - прочитать все строки в список.

`write(s)` - записать строку или байты.

`writelines(seq)` - записать последовательность строк.

`flush()` - принудительно сбросить буфер на диск.

### 6. Менеджер контекста `with`

Лучший способ работать с файлами:

```python
with open("data.txt", "r", encoding="utf-8") as f:
    text = f.read()
```

После выхода из блока файл закроется автоматически, даже если внутри произошла ошибка.

### 7. Текстовый и бинарный режимы

В текстовом режиме Python работает со строками `str`.

```python
with open("data.txt", "w", encoding="utf-8") as f:
    f.write("hello\n")
```

В бинарном режиме Python работает с `bytes` и `bytearray`.

```python
with open("data.bin", "wb") as f:
    f.write(b"\x00\x01\x02")
```

### 8. `print(..., file=f)`

Писать в файл можно через `print`:

```python
with open("log.txt", "w", encoding="utf-8") as f:
    print("Hello", "world", sep=", ", file=f)
```

Важные параметры `print`:

`sep` - разделитель между объектами.

`end` - что печатать в конце, по умолчанию `\n`.

`file` - объект с методом `write`.

`flush` - принудительно сбрасывать буфер.

### 9. Сериализация и десериализация

Сериализация - перевод структуры данных в последовательность байтов или текстовый формат для хранения или передачи.

Десериализация - обратное восстановление объекта из сохраненного представления.

Используется для:

- сохранения объектов в файлы;
- обмена данными между программами;
- передачи данных по сети;
- кеширования.

### 10. `pickle`

`pickle` сериализует Python-объекты в бинарный формат.

```python
import pickle

data = {"name": "Ann", "age": 20}

with open("data.pkl", "wb") as f:
    pickle.dump(data, f)

with open("data.pkl", "rb") as f:
    restored = pickle.load(f)
```

Функции:

`pickle.dump(obj, file)` - записать объект в файл.

`pickle.dumps(obj)` - вернуть байты.

`pickle.load(file)` - прочитать объект из файла.

`pickle.loads(bytes_object)` - восстановить объект из байтов.

Исключения: `PickleError`, `PicklingError`, `UnpicklingError`.

Недостатки `pickle`:

- формат специфичен для Python;
- нельзя безопасно загружать данные из недоверенного источника, потому что при десериализации может выполниться вредный код;
- не лучший вариант для долговременного хранения между разными версиями приложений.

### 11. `cloudpickle`

`cloudpickle` расширяет возможности `pickle`. Он умеет сериализовать, например, lambda-функции и объекты, определенные в интерактивной среде.

Используется в распределенных вычислениях, когда Python-код отправляется на удаленные машины.

Но он тоже не предназначен для долговременного хранения объектов и обычно требует совместимых версий Python.

### 12. Форматы данных: CSV, JSON, YAML

CSV - табличные данные.

JSON - обмен структурированными данными, хорошо подходит для API.

YAML - человекочитаемые конфигурации.

XML и INI тоже встречаются, но в лекции основной акцент на CSV, JSON, YAML.

### 13. CSV

CSV представляет таблицу: одна строка файла - одна строка таблицы, значения разделяются delimiter-символом, часто запятой.

Пример:

```csv
name,age
Ann,20
Bob,21
```

Чтение:

```python
import csv

with open("users.csv", newline="", encoding="utf-8") as f:
    reader = csv.reader(f)
    for row in reader:
        print(row)
```

Запись:

```python
import csv

with open("users.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.writer(f)
    writer.writerow(["name", "age"])
    writer.writerow(["Ann", 20])
```

`DictReader` читает строки как словари:

```python
with open("users.csv", newline="", encoding="utf-8") as f:
    reader = csv.DictReader(f)
    for row in reader:
        print(row["name"])
```

`DictWriter` пишет словари:

```python
with open("users.csv", "w", newline="", encoding="utf-8") as f:
    writer = csv.DictWriter(f, fieldnames=["name", "age"])
    writer.writeheader()
    writer.writerow({"name": "Ann", "age": 20})
```

Диалект CSV - набор правил: разделитель, символ кавычек, правила экранирования.

Важные параметры:

`delimiter` - разделитель.

`quotechar` - символ кавычек.

`quoting` - когда ставить кавычки.

Константы:

`csv.QUOTE_ALL` - кавычки у всех полей.

`csv.QUOTE_MINIMAL` - кавычки только где нужно.

`csv.QUOTE_NONNUMERIC` - нечисловые поля в кавычках, числовые читать как `float`.

`csv.QUOTE_NONE` - не использовать кавычки.

### 14. JSON

JSON - текстовый формат обмена данными. Он похож на структуры Python:

JSON object -> Python `dict`.

JSON array -> Python `list`.

string -> `str`.

number -> `int` или `float`.

true/false -> `True`/`False`.

null -> `None`.

```python
import json

data = {"name": "Ann", "age": 20}

s = json.dumps(data, ensure_ascii=False, indent=2)
obj = json.loads(s)
```

`json.dumps(obj)` - объект Python в JSON-строку.

`json.loads(s)` - JSON-строку в Python-объект.

Для файлов:

```python
with open("data.json", "w", encoding="utf-8") as f:
    json.dump(data, f, ensure_ascii=False, indent=2)

with open("data.json", "r", encoding="utf-8") as f:
    data = json.load(f)
```

Если JSON некорректен, возникает `json.JSONDecodeError`.

### 15. YAML

YAML - формат для удобного ручного описания данных и конфигураций.

Особенности:

- структура задается отступами;
- нельзя использовать табы для отступов;
- комментарии начинаются с `#`;
- строки часто можно писать без кавычек;
- списки обозначаются `-`;
- словари пишутся как `ключ: значение`.

Пример:

```yaml
server:
  host: localhost
  port: 8000
users:
  - name: Ann
    age: 20
  - name: Bob
    age: 21
```

В Python обычно используют PyYAML:

```python
import yaml

with open("config.yml", encoding="utf-8") as f:
    config = yaml.safe_load(f)
```

Важно использовать `safe_load`, а не небезопасный `load` без нужного loader.

### 18. `zipfile`

`zipfile` позволяет читать и создавать ZIP-архивы.

```python
from zipfile import ZipFile, ZIP_DEFLATED

with ZipFile("archive.zip", "w", compression=ZIP_DEFLATED) as z:
    z.write("data.txt", arcname="data.txt")
```

Режимы:

`r` - читать архив.

`w` - создать архив заново.

`a` - добавить в существующий архив.

Типы сжатия:

`ZIP_STORED` - без сжатия.

`ZIP_DEFLATED` - обычное zip-сжатие.

`ZIP_BZIP2`, `ZIP_LZMA` - другие алгоритмы.

Методы:

`namelist()` - список имен.

`infolist()` - информация обо всех файлах.

`getinfo(name)` - информация об одном файле.

`read(name)` - прочитать файл из архива как байты.

`open(name)` - открыть файл внутри архива.

`extract(name)` - извлечь один файл.

`extractall(path)` - извлечь все.

`printdir()` - вывести содержимое.

`ZipInfo` хранит `filename`, `date_time`, `compress_type`, `compress_size`, `file_size`.

## Лекции 10-11. Модули, стандартная библиотека, регулярные выражения, визуализация

### 1. Модуль

Модуль - файл `.py`, содержащий функции, классы и данные. Имя файла становится именем модуля.

Модули нужны для:

- логической группировки кода;

Пакет - папка с модулями и подпакетами. Классически содержит `__init__.py`, который отвечает за инициализацию пакета.

Пример:

```text
my_package/
```

Импорт:

```python
import my_package.module1
from my_package import module1
```

### 3. Классификация модулей

Встроенные - встроены в интерпретатор, часто написаны на C. Примеры: `sys`, `math`, `time`.

Стандартная библиотека - поставляется вместе с Python. Примеры: `json`, `os`, `csv`, `datetime`.

Сторонние - устанавливаются из PyPI через `pip`. Примеры: `numpy`, `matplotlib`, `django`.

Пользовательские - модули, которые создает сам программист.

Установка стороннего пакета:

```bash

```

### 4. Способы импорта

Рекомендуемый способ:

```python
import math
math.sin(1)
```

Импорт с псевдонимом:

```python
import numpy as np
```

Импорт конкретных объектов:

```python
from math import sin, radians
```

Импорт всего:

```python
from math import *
```

Последний вариант обычно не рекомендуется: легко получить конфликт имен и непонятно, откуда взялась функция.

### 5. Поиск модулей и `sys.path`

При импорте Python ищет модуль:

1. среди встроенных;
2. в текущей директории и путях `sys.path`;
3. в путях установки Python и `site-packages`.

Если модуль не найден, будет `ModuleNotFoundError`.

Если найден, но не загрузился, может быть `ImportError`.

Важно не называть свои файлы как стандартные модули, например `json.py`, иначе Python может импортировать ваш файл вместо стандартного `json`.

### 6. Специальные атрибуты модулей

`__name__` - имя модуля. Если файл запущен напрямую, значение `"__main__"`.

`__doc__` - строка документации.

`__file__` - путь к файлу модуля.

Частый шаблон:

```python
def main():
    print("program started")

if __name__ == "__main__":
    main()
```

Так код запускается только при прямом запуске файла, но не при импорте.

Дополнительные атрибуты: `__author__`, `__version__`, `__license__` и т.д.

### 7. `.pyc` и `__pycache__`

Python компилирует импортируемые модули в байт-код `.pyc` и хранит его в `__pycache__`.

Это ускоряет загрузку, но не делает выполнение кода быстрее. `.pyc` зависит от версии Python и платформы. Его можно декомпилировать, поэтому это не надежный способ скрыть исходный код.

### 16. Модуль `re`

Основные функции:

`re.finditer(pattern, string)` - итератор match-объектов.

`re.split(pattern, string)` - разбиение по шаблону.

## Лекция 12. Сети, HTTP и Django

### 1. Компьютерная сеть

Компьютерная сеть - система устройств и каналов связи для передачи данных между компьютерами и программами.

Сеть должна решать вопросы:

- как найти получателя;
- как доставить данные;
- как проверить ошибки;
- как понять, какой программе на узле предназначены данные;
- как представить данные в понятной форме.

### 2. Открытые системы и модель OSI

Открытая система строится на открытых спецификациях, то есть ее можно расширять и соединять с другими системами.

Модель OSI делит сетевое взаимодействие на 7 уровней:

1. Физический - сигналы, кабели, радиоканал, электрический/оптический уровень.
2. Канальный - кадры, MAC-адреса, контроль ошибок на участке сети.
3. Сетевой - логическая адресация и маршрутизация, например IP.
4. Транспортный - передача между программами, надежность, порты, TCP/UDP.
5. Сеансовый - установление и управление сеансами.
6. Представления - форматы данных, кодировки, шифрование/сжатие.
7. Прикладной - протоколы приложений: HTTP, FTP, SMTP, DNS и т.д.

Уровни образуют стек: верхние используют нижние. Это называется инкапсуляцией.

### 3. TCP/IP

TCP/IP - стек протоколов, на котором работает Интернет.

Уровни TCP/IP приблизительно соответствуют OSI:

- уровень сетевых интерфейсов;
- уровень межсетевого взаимодействия;
- транспортный уровень;
- прикладной уровень.

Сетевое взаимодействие требует минимум двух участников: отправителя и получателя.

Полный идентификатор участника можно представить так:

```text
сетевой адрес : транспортный протокол : порт
```

Пример:

```text
192.168.0.99:TCP:8080
```

### 4. IP-адреса

IP работает на сетевом уровне и решает две главные задачи:

- адресация узлов;
- доставка пакетов между сетями.

Данные передаются порциями: пакетами, или датаграммами.

IPv4 - 32-битный адрес, обычно записывается четырьмя числами:

```text
192.168.0.1
```

IPv6 - 128-битный адрес.

Специальные IPv4-адреса:

`0.0.0.0` - неопределенный адрес или источник.

`255.255.255.255` - широковещательный адрес.

`127.0.0.1` - localhost, локальная машина.

`10.x.x.x`, `172.16.x.x` - `172.31.x.x`, `192.168.x.x` - частные адреса локальных сетей.

### 5. UDP и TCP

UDP - простой транспортный протокол. Он быстро отправляет датаграммы, но не гарантирует доставку.

Плюсы UDP:

- быстрее;
- меньше служебных данных;
- поддерживает широковещательную рассылку.

Минусы UDP:

- нет гарантии доставки;
- нет гарантии порядка;
- приложение само должно решать вопросы потерь.

TCP - протокол надежной доставки, ориентированный на соединение.

TCP:

- устанавливает соединение;
- разбивает данные на сегменты;
- подтверждает получение;
- повторяет потерянные сегменты;
- собирает данные в правильном порядке.

Плюсы TCP:

- надежность;
- контроль порядка;
- удобен для большинства прикладных протоколов.

Минусы:

- медленнее UDP;
- требует установления соединения;
- нет широковещательной рассылки.

Примеры:

HTTP - TCP 80.

HTTPS - TCP 443.

SMTP - TCP 25.

FTP - TCP 21.

DNS часто использует UDP 53.

### 6. DNS и доменные имена

IP-адреса неудобны человеку, поэтому используют символьные имена: `example.com`.

DNS - Domain Name System - иерархическая система доменных имен. Она переводит доменное имя в IP-адрес и иногда обратно.

Алгоритм может включать:

- проверку локальной таблицы;
- обращение к DNS-кэшу;
- запрос к DNS-серверам.

### 7. URI, URL, URN

URI - универсальный идентификатор ресурса.

URL - локатор ресурса, то есть где и как получить ресурс.

URN - имя ресурса, не обязательно указывающее место получения.

Структура URL:

```text
scheme://host:port/path?query#fragment
```

Пример:

```text
http://www.example.com/site/map.html
```

### 8. Порты

Порт - число от 1 до 65535, которое помогает направить сетевые данные конкретной программе на компьютере.

Диапазоны:

`0-1023` - системные, общеизвестные.

`1024-49151` - зарегистрированные.

`49152-65535` - динамические, временные.

Сервер слушает известный порт, клиент часто использует временный свободный порт.

### 9. HTTP

HTTP - Hypertext Transfer Protocol, прикладной протокол для передачи гипертекста и других ресурсов.

HTTP работает по модели запрос-ответ:

- клиент отправляет запрос;
- сервер возвращает ответ.

Сервер не начинает HTTP-диалог сам, он отвечает на запросы клиента.

HTTPS - HTTP поверх защищенного канала TLS.

### 10. Структура HTTP-сообщения

HTTP-сообщение содержит:

1. стартовую строку;
2. заголовки;
3. пустую строку;
4. тело, если оно есть.

Запрос:

```http
GET /index.html HTTP/1.1
Host: example.com
```

Ответ:

```http
HTTP/1.1 200 OK
Content-Type: text/html
```

### 11. HTTP-методы

`GET` - получить ресурс.

`POST` - отправить данные на обработку.

`PUT` - заменить или создать ресурс.

`PATCH` - частично изменить ресурс.

`DELETE` - удалить ресурс.

`HEAD` - как GET, но без тела ответа.

`OPTIONS` - узнать доступные возможности.

### 12. HTTP-коды состояния

`1xx` - информационные.

`2xx` - успех.

`3xx` - перенаправления.

`4xx` - ошибка клиента.

`5xx` - ошибка сервера.

Примеры:

`200 OK` - успешно.

`301 Moved Permanently` - ресурс переехал навсегда.

`400 Bad Request` - плохой запрос.

`401 Unauthorized` - требуется аутентификация.

`403 Forbidden` - доступ запрещен.

`404 Not Found` - ресурс не найден.

`500 Internal Server Error` - внутренняя ошибка сервера.

`502 Bad Gateway` - ошибка шлюза.

### 13. HTTP-заголовки

Заголовки записываются так:

```http
Header-Name: value
```

Они описывают параметры передачи, тело сообщения, клиента, сервер, кеширование, язык, авторизацию и т.д.

Примеры:

`Host` - имя хоста.

`User-Agent` - информация о клиенте.

`Accept-Language` - предпочитаемый язык.

`Content-Type` - тип тела.

`Content-Length` - длина тела.

`Location` - адрес для перенаправления.

### 14. Динамические сайты и серверное программирование

Статические ресурсы - файлы, которые сервер отдает как есть: CSS, JS, картинки, PDF.

Динамический сайт формирует часть ответа во время запроса: достает данные из базы, подставляет их в HTML-шаблон, учитывает пользователя.

Серверные языки и платформы: Python, PHP, Ruby, C#, Node.js.

Веб-фреймворк - набор инструментов для типовых задач веб-разработки:

- маршрутизация URL;
- обработка запросов;
- работа с базой данных;
- шаблоны;
- формы;
- сессии;
- аутентификация;
- безопасность;
- форматирование ответов HTML/JSON/XML.

### 15. Django

Django - Python-фреймворк для серверной веб-разработки.

Он ориентирован на полноту, безопасность, масштабируемость, переносимость и удобство сопровождения.

Архитектурный паттерн Django - MVT:

Model - данные и логика работы с базой.

View - функция или класс, который принимает запрос и возвращает ответ.

Template - HTML-шаблон с местами для подстановки данных.

Это близко к MVC, но в терминологии Django контроллерская часть частично находится внутри самого фреймворка и URL-маршрутизатора.

### 16. Создание проекта Django

Команды:

```bash
django-admin startproject project_name
python manage.py runserver
```

`manage.py` - утилита управления проектом.

После запуска сервер обычно доступен:

```text
http://127.0.0.1:8000/
http://localhost:8000/
```

Файлы проекта:

`__init__.py` - папка считается пакетом Python.

`settings.py` - настройки проекта: база данных, приложения, пути, безопасность.

`urls.py` - маршруты URL.

`wsgi.py` - точка входа для WSGI-серверов при развертывании.

`asgi.py` - точка входа для асинхронного ASGI.

### 17. Приложения Django

Проект может состоять из нескольких приложений.

Создание:

```bash
python manage.py startapp hello
```

Файлы приложения:

`migrations/` - миграции базы данных.

`admin.py` - регистрация моделей в админке.

`apps.py` - конфигурация приложения.

`models.py` - модели данных.

`tests.py` - тесты.

`views.py` - обработчики запросов.

Приложение нужно добавить в `INSTALLED_APPS` в `settings.py`.

### 18. View и URL-маршрутизация

Простейший view:

```python
from django.http import HttpResponse

def index(request):
    return HttpResponse("Hello")
```

Маршрут:

```python
from django.urls import path
from hello import views

urlpatterns = [
    path("", views.index, name="home"),
]
```

`path` удобен для простых маршрутов:

```python
path("user/<name>/", views.user)
path("user/<int:age>/", views.user)
```

`re_path` использует регулярные выражения:

```python
re_path(r"^user/(?P<name>\D+)/(?P<age>\d+)", views.user)
```

`TemplateView` может отдавать шаблон без собственной view-функции:

```python
from django.views.generic import TemplateView

path("about/", TemplateView.as_view(
    template_name="about.html",
    extra_context={"header": "О сайте"},
))
```

### 19. Шаблоны Django

Шаблон - HTML-файл с Django template language.

Настройки шаблонов в `settings.py`:

`BACKEND` - движок шаблонов.

`DIRS` - список папок для поиска шаблонов.

`APP_DIRS` - искать ли шаблоны внутри приложений в папке `templates`.

`OPTIONS` - дополнительные настройки.

Render:

```python
from django.shortcuts import render

def index(request):
    return render(request, "index.html", {"name": "Ann"})
```

В шаблоне:

```html
<h1>Hello, {{ name }}</h1>
```

Наследование шаблонов:

```html
{% extends "base.html" %}

{% block content %}
  <h1>Main page</h1>
{% endblock %}
```

Подключение одного шаблона в другой:

```html
{% include "banner.html" %}
{% include "banner.html" with tutorial="Python" %}
```

### 20. Static-файлы

Static - CSS, JS, изображения.

В шаблоне:

```html
{% load static %}
<link rel="stylesheet" href="{% static 'css/styles.css' %}">
<img src="{% static 'images/forest.jpg' %}" alt="forest">
```

### 21. Модели Django

Модель - Python-класс, который описывает таблицу базы данных.

```python
from django.db import models

class Genre(models.Model):
    name = models.CharField(max_length=100)
```

Частые параметры полей:

`help_text` - подсказка для формы.

`verbose_name` - человекочитаемое имя поля.

`default` - значение по умолчанию.

`null=True` - хранить `NULL` в базе.

Важно: для `CharField` пустое значение обычно хранится как пустая строка, а не `NULL`.

Типы полей:

`CharField` - короткая строка, нужен `max_length`.

`TextField` - длинный текст.

`IntegerField` - целое число.

`DateField`, `DateTimeField` - дата и дата-время.

`EmailField` - email с проверкой.

`FileField`, `ImageField` - загрузка файлов и изображений.

`AutoField` - автоинкрементный ключ.

Связи:

`ForeignKey` - один-ко-многим.

`ManyToManyField` - многие-ко-многим.

`on_delete` определяет поведение при удалении связанной записи.

### 22. Миграции, база и админка

Миграция - способ привести структуру базы данных к описанию моделей.

Команды:

```bash
python manage.py makemigrations
python manage.py migrate
```

По умолчанию часто используется `db.sqlite3`.

Регистрация модели в админке:

```python
from django.contrib import admin
from .models import Genre

admin.site.register(Genre)
```

Создание суперпользователя:

```bash
python manage.py createsuperuser
python manage.py runserver
```

Админка:

```text
http://127.0.0.1:8000/admin
```

При вводе пароля в терминале символы обычно не отображаются. Это нормально.
